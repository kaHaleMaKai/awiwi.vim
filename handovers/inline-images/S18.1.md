# S18.1 — `asset.resolve_image_link` (backend-agnostic groundwork)

status: done
module: `lua/awiwi/asset.lua`
tests: `tests/asset_spec.lua`

## What this is

Extracts the image-path resolution logic that is currently inlined at
`lua/awiwi/init.lua:559-566` (the `elseif t == "image" then` branch of
`M.open_link`) into a new, pure, standalone public function on `awiwi.asset`.
This unit does **not** touch `init.lua` — that's S18.2's job. This unit is
also backend-agnostic: it has nothing to do with kitty/snacks.nvim rendering,
it's just the "given an `image` link target, what asset file does it point
at" resolution step, factored out so both `init.lua` (link-opening) and the
future `awiwi.img` module (S19.1, inline rendering) can share it.

## Signature

```lua
M.resolve_image_link(target: string) -> string|nil
```

Location: `lua/awiwi/asset.lua`, right before `M.open_asset_sink` (after
`M.open_asset_by_name`, before the "fzf/telescope-picker sink shape" comment
block), around line 439 post-edit.

## Contract

Given `target` (an `image`-type link's raw target string, as read from a
markdown link `![...](target)` — see `init.lua:559-566` for the shape it's
called with):

1. **Happy path** — `target` is an absolute path (`pathlib.is_absolute`,
   i.e. starts with `/`) whose immediate parent directory component matches
   `^%d%d%d%d%-%d%d%-%d%d$` (a `YYYY-MM-DD` date), e.g.
   `/assets/2024-03-05/pic.png` or `/x/2024-03-05/pic.png` (the date-parent
   check does **not** care what comes before it in the path — only the last
   path component before the filename matters). Returns
   `pathlib.join(M.deps.get_asset_subpath(), "2024", "03", "05", "pic.png")`.
2. **Relative target** (no leading `/`, e.g. `2024-03-05/pic.png` or
   `./img.png`) → `nil`.
3. **`http://`/`https://` URL** → `nil` (these fail the `is_absolute` check
   too since they don't start with `/`, so this case is subsumed by the
   absolute-path gate rather than needing a dedicated regex — noted here so
   nobody "fixes" it into a separate branch later).
4. **Absolute target whose parent dir is not a date** (e.g. `/tmp/pic.png`)
   → `nil`.
5. Goes through the existing `M.deps.get_asset_subpath()` seam (same one
   `M.get_asset_path` and `M.get_all_asset_files` already use) instead of
   reading `vim.g.awiwi_home` directly — a test-injected override of
   `asset.deps.get_asset_subpath` is honored.

Pure function: no side effects, no filesystem access (`vim.fn.fnamemodify`
is pure string manipulation, doesn't stat the path), no `vim.g` reads.

## Implementation

```lua
function M.resolve_image_link(target)
  if not pathlib.is_absolute(target) then
    return nil
  end
  local dir = vim.fn.fnamemodify(target, ":h:t")
  local y, m, d = dir:match("^(%d%d%d%d)%-(%d%d)%-(%d%d)$")
  if not y then
    return nil
  end
  local fname = vim.fn.fnamemodify(target, ":t")
  return pathlib.join(M.deps.get_asset_subpath(), y, m, d, fname)
end
```

## Tests added (`tests/asset_spec.lua`, `describe("asset.resolve_image_link", ...)`)

- `resolves an absolute target whose parent dir is a YYYY-MM-DD date` (item 1,
  covers both `/assets/2024-03-05/pic.png` and `/x/2024-03-05/pic.png`)
- `returns nil for a relative target` (item 2)
- `returns nil for an http(s) URL` (item 3)
- `returns nil when the absolute target's parent dir is not a date` (item 4)
- `resolves through the M.deps.get_asset_subpath() seam` (item 5, overrides
  `asset.deps.get_asset_subpath` to `/custom/assets` and checks the returned
  path is rooted there)

All use the existing `with_home(fn)` test-isolation helper already in the
spec file (pointer `vim.g.awiwi_home` at a scratch tempdir, restored after).
