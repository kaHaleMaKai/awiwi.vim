# S31.1 — index (home) page branding + "today" quick link

## Stakeholder asks, adopted interpretation

1. Root `h1.page-title` "g:awiwi_home" → `awīwī /awi:ˈi:/` (verbatim, macrons + IPA-ish
   string kept as given).
2. Sub text directly under it, italic: `ʻAʻole lohi nohoʻi.` (verbatim ʻokina chars).
3. Quick link right of the "home" breadcrumb, `|`-separated. **Interpretation (mine,
   adopted for this unit)**: a link labeled `today` that resolves to today's journal
   entry, index page only.

## How "today" resolves

Uses the **server's** notion of today, not the client clock: `DirPage.svelte` already
fetches `GET /api/meta` once per mount for the journal-month "current day" highlight
(`today` state, populated from `MetaPayload.today`, a `YYYY-MM-DD` string). The quick
link reuses that same value: `href = /journal/${today}`.

Verified before implementing (`server/src/awiwi/content.py:22-42`,
`server/src/awiwi/docs.py:286-314`): `build_journal_payload` runs `date_str` through
`content.parse_date`, which special-cases the literal string `"today"` — so
`/api/journal/today` genuinely resolves server-side. I did **not** use that literal
path though: `JournalPage.svelte:86` renders `journalTitle(date)` from the raw route
param verbatim (not the resolved date from the payload), and `journalTitle()` returns
non-ISO input unchanged — so a `/journal/today` link would land on a page whose H1
reads literally "today" instead of the formatted date. `JournalPage.svelte` is outside
this unit's boundary (owned elsewhere), so I used the already-available, already-ISO
`today` string from `getMeta()` instead: `/journal/2026-07-15`-shaped links that render
correctly with zero cross-boundary changes. Client-side router matcher
(`router.svelte.ts`) accepts any string as `:date` — no router changes needed either
way.

## Changes

- `src/lib/components/DirPage.svelte`:
  - `ROOT_TITLE = "awīwī /awi:ˈi:/"` replaces the `"g:awiwi_home"` title fallback.
  - `ROOT_SUBTITLE = "ʻAʻole lohi nohoʻi."` rendered as `<p class="page-subtitle">`
    directly under the `h1`, gated `{#if isRoot}` (non-root dirs unchanged).
  - Breadcrumb-setting logic moved out of the `getDir().then()` callback into its own
    `$effect` that reacts to both `dir` and `today` (previously only fired once, on the
    dir fetch resolving — `today` arrives from a separate, independently-timed
    `getMeta()` fetch, so a single one-shot effect could race and miss it). For the
    root path, once `today` has resolved, it sets breadcrumbs to
    `[{name:"home",target:"/"}, {name:"today", target:"/journal/${today}"}]`; every
    other path/case behaves exactly as before (unchanged `dir.breadcrumbs` or the
    single-crumb root fallback while `today` hasn't loaded yet).
- `src/lib/components/Breadcrumbs.svelte`: detects that exact `[home, today]` shape
  (`crumbs.length === 2 && crumbs[0].name === "home" && crumbs[0].target === "/"`) and
  renders it as two real links joined by a literal `|` `.sep`, instead of the normal
  `›`-separated trail whose last crumb is plain (non-link) "current page" text. All
  other crumb shapes (every other route) go through the untouched original branch.
- `src/app.css`: added `.page-subtitle { font-style: italic; color: var(--text-muted); }`
  next to `.page-title`. No changes to `.breadcrumbs .sep` — it already renders plain
  `color: var(--border-strong)` text regardless of glyph, so `|` picks up the same
  style as `›` for free.

No pure-logic module was extracted (the "today" URL is a one-line template string
reusing the existing `today` state, not separable behavior) so no new vitest spec was
added; the store/component changes are covered by the existing gates below.
