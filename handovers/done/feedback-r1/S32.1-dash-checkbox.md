# S32.1 — dash-bullet checkbox rendering

## Root cause

`server/src/awiwi/mdrender.py:51` `_CHECKBOX_LINE_RE` matched only `* [ ]`/`* [x]`
asterisk-bullet lines. Stakeholder's notes use `- [ ]`/`- [x]` dash bullets, which fell
through as literal text (`_INLINE_MARKUP_RE`/ordinal substitution only, no checkbox
branch), so the SPA's `input.awiwi-checkbox` enhancer never saw them.

`server/src/awiwi/checkbox.py` had the same `* `-only assumption in three places:
`_CHECKBOX_ITEM_RE` (used by `hash_line` to strip the box before hashing) and
`_CHECKBOX_PREFIX_RE` (used by `toggle_checkbox` to locate/flip the box). A dash line's
hash previously included the box glyph (since it never matched `_CHECKBOX_ITEM_RE`), and
`toggle_checkbox` raised `NotACheckboxLineError` for any dash line.

## Changes

- `server/src/awiwi/mdrender.py:51` — `_CHECKBOX_LINE_RE`:
  `^(\s*\* )(\[[x ]\])( .*$)` → `^(\s*[*-] )(\[[x ]\])( .*$)`. `+ [ ]` intentionally
  excluded per spec (out of scope).
- `server/src/awiwi/checkbox.py` — **yes, checkbox.py needed changes**:
  - `_CHECKBOX_ITEM_RE`: `\s*\* \[[x ]\] ` → `\s*[*-] \[[x ]\] `
  - `_CHECKBOX_PREFIX_RE`: `(\s*\* \[)([ x])` → `(\s*[*-] \[)([ x])`
  - Docstrings on `hash_line`, `toggle_checkbox`, `NotACheckboxLineError` updated to
    mention the dash-bullet case.
  - Existing asterisk-line hashes are unchanged (golden values in
    `TestHashLine` untouched, still pass) — the change only widens the alternation,
    doesn't touch the asterisk path.
