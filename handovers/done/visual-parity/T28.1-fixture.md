# T28.1 — visual-parity fixture tree + page-pair manifest

Zero-context handover: read this before touching `server/tests/visual/fixture/home/**` or
`server/tests/visual/pairs.json`, or before writing T28.2's `shoot.mjs` / T28.3's comparator
prompts.

## What this is

A committed notes tree (`server/tests/visual/fixture/home/`) whose rendered SPA output
reproduces, as closely as the real render pipeline allows, the sample content baked into the
12 static T22 mockup HTML files (`mockups/*.html`). Plus `server/tests/visual/pairs.json`, the
manifest T28.2's screenshot harness and T28.3's comparators consume verbatim.

Boot for manual poking: `cd server && AWIWI_HOME=$PWD/tests/visual/fixture/home uv run uvicorn
awiwi.app:app --port 5824` (port 5824 so the user's own 5823 instance is untouched). Serves the
**built dist** (`server/frontend/dist`, `StaticFiles`) — rebuild (`npm run build`) picks up
frontend changes without restarting uvicorn.

## Render-pipeline facts this fixture leans on (don't re-derive)

- **Redaction** (`mdrender.py:_filter_body`): `!!redacted` splits the line into `before`
  (becomes the `.redacted` span, HTML-escaped, revealed since `embed_redacted=True` by default
  when `AWIWI_ALLOW_REMOTE` is unset and the request is localhost) and `rest` (becomes a
  `(cause: ...)` suffix, UNLESS the marker is the last thing on the line, in which case the
  placeholder is the bare `" --- redacted --- "` with no cause). The journal fixture puts the
  marker at the end of the physical line containing the fake credential, then continues the
  visible "do not paste..." clause on the *next* source line of the same paragraph (no blank
  line between them) — `Nl2BrExtension` is enabled, so this renders as one `<p>` with a forced
  `<br>` between the pill and the rest of the sentence. This is the closest achievable
  approximation of the mockup's inline mid-sentence pill; see the journal exclusion entry.
- **`_SECRET_RE`** (`\b(secret|credential)s?\b$`) matches `rotate-credentials` (the file stem),
  setting `is_secret=True`, but content is only blanked `if is_secret and not is_localhost`. All
  verification here is against `127.0.0.1`, so content renders in full — this only matters if a
  comparator or screenshot script somehow runs against a non-localhost `Host` header.
- **`.txt` language sniffing**: no extension→language mapping exists for `.txt`;
  `guess_language` sniffs a vim modeline (`vim: ft=<lang>`) anywhere in the file text.
  `deploy-notes.txt` has an extra first line, `# vim: ft=bash.` (trailing period so the regex's
  required trailing whitespace/dot always matches, even at EOF), making the file **15 lines**,
  not the mockup's 14.
- **No filename toolbar on markdown-embedded code fences**: `shiki.ts`/`copyButtons.ts` add
  only a floating, unlabeled "Copy" button to `<pre>` blocks inside markdown bodies — there is
  no per-block filename header like the mockup's `resolve_image_link.py` toolbar anywhere in
  the real component tree. Only `TextFileView.svelte` (whole-file asset view) has a real
  toolbar, and its text is `{watch_path} · {lineCount} lines · {lang}` — no "Copy file" labeled
  button either (same unlabeled floating "Copy").
- **Todo page** (`TodoPage.svelte`): renders `journal/todos.md` as plain markdown, hardcodes an
  `<h1>` of "Todos" (not the mockup's "Todo"). No filter chips, no due-date/journal-ref badges,
  no todo-row grouping metadata — `## Overdue`/`## Today`/`## Upcoming` are just headings with
  checkbox lists underneath, structurally identical to any other markdown page.

## Per-page exclusion list (feed verbatim into T28.3 comparator prompts)

### journal / journal-light — `/journal/2026-07-14`

- Shiki token colors (real, client-rendered) vs mockup's hand-written `.tok-*` spans — token
  *boundaries* may legitimately differ since real highlighting is grammar-driven, not
  hand-placed; compare only the color *palette*/weight, not exact per-character spans.
- Code fence has **no filename toolbar** at all in the real render (mockup shows
  `resolve_image_link.py` in a `.code-toolbar`) — this is not a styling gap, there is no
  component for it. Ignore that whole band; only compare the `<pre>` body + floating Copy
  button.
- Mermaid: real page renders an actual SVG diagram client-side (mermaid.js); fixture's fence
  content (`flowchart TD` with 3 nodes) will produce a real, differently-shaped diagram, not the
  mockup's placeholder box + "mermaid diagram renders here" caption. Compare only the
  container's border/background/spacing, never the diagram artwork itself.
- Redacted line: real output wraps the **entire sentence prefix** ("Spent most of the day on
  the viewer redesign. The staging credentials are sk-live-9f2a...c001") in `.redacted`, not
  just the fake credential token, followed by literal `" --- redacted --- "` text and a forced
  line break before "— do not paste these tokens into the shared channel again." Mockup shows a
  compact inline pill (`sk-live-9f2a...c001`) mid-sentence with no line break and no residual
  text. Ignore pill *extent*/wording and the line-break; only compare pill color/shape/font.
- `ws-dot` state/pulse animation — always exclude (documented globally, harness kills
  animations anyway).
- Task list DOM shape: real checkboxes render as `<ul><li><input>...<label>...</label></li>`
  (legacy `awiwi-checkbox` markup), not the mockup's `.task-item` flex rows. Compare checkbox
  color/checked-state styling, not row layout/spacing, unless `app.css` has specifically been
  updated to reproduce the row look via `li`/`ul` selectors.
- TOC rail / sync-status legend are static decorative mockup content vs the SPA's real
  `doc.toc`-driven rail — the *headings listed* should match (Morning/Work/Standup
  notes/Reading/Tasks/Diagram) but any hover/active-state styling nuance is fair game to diff,
  el count/order is not.

### dir-root — `/`

- "Recent files" section: **no backing API concept** (feature-excluded per T28 user decision) —
  ignore the entire section; the SPA route may not render it at all.
- File sizes/mtimes ("edited just now", "2 hours ago", "4.1 MB"): no such metadata exists in
  the real dir payload — ignore any such text entirely.
- Row *counts*/*names* should match (journal/, assets/, recipes/ with fixture-derived
  descriptions) — only the presentational chrome is excluded, not structural presence of the
  three top-level rows.

### dir-month — `/dir/journal/2026/07`

- Week-banding into "Week 27"/"Week 28" groups and day-summary/word-count columns are
  mockup-only decoration with no API backing (`build_dir_payload` returns bare
  name/relpath/is_dir/doc_type) — ignore entirely; only compare that the real page lists a
  flat/grouped-by-week set of day rows with correct *dates*.
- Mockup shows Jul 08 as "No entry" (day skipped, greyed) and Jul 15/16/17 as "is-future / Not
  written yet" placeholder rows. The real listing only returns rows for files that exist: our
  fixture has **no** 07-08 file (so it won't appear at all, not greyed-out — a present/absent
  difference, not a style difference) and **does** have a real 07-15 file (nav-stub, so it
  *will* appear as a normal row, not a greyed "future" row). Do not flag either as a parity bug.
- Jul 16/17 exist in the mockup only — no fixture files for them, so those rows are simply
  absent from the real page. Not a bug.

### todo — `/todo`

- Filter chips (Open/Done/All) and due-date/journal-ref badges (`due Jul 10`,
  `journal/2026-07-08`, etc.): **no backing feature** — `TodoPage.svelte` has no chip/badge
  markup at all, plain markdown only. Ignore entirely.
- Page `<h1>` text: real page hardcodes "Todos", mockup shows "Todo" — this IS a legitimate
  wording diff worth flagging if the intent is to match, but per current source it's
  intentional (see `TodoPage.svelte` comment) — flag as low severity / confirm before fixing.
- Task-row layout (icon/checkbox/badge cluster) — real render is a plain checkbox list, same
  DOM-shape caveat as journal's Tasks section above.

### search / search-light — `/search?q=token`

- Search-hit **text content** will not literally match the mockup's invented snippet prose
  (e.g. mockup's fictional journal hit reads "Spent most of the day on the viewer redesign. The
  staging credentials are token" — not real fixture text). Our real hits come from `rg`
  matching actual fixture lines: expect journal hits on "— do not paste these tokens..." and
  "Design checkpoint today — Noir-Deco token review." (different lines/wording than mockup).
  `recipes/ops/rotate-credentials.md` line 3 WAS deliberately authored to literally match the
  mockup's snippet ("Rotate the staging token every 30 days via the admin console.") — that one
  hit can be compared for exact text; none of the others should be.
- Header search input: mockup hardcodes `value="redacted OR token"` — a fixed illustrative
  quirk, not the actual query (`token` alone). Ignore the exact input value text; only compare
  input chrome/placeholder styling.
- `ws-dot is-reconnecting` state shown in this mockup specifically — exclude per the global
  ws-dot rule (harness kills animation, and the real page's connection state is whatever the
  live socket reports, not deterministically "reconnecting").
- Result grouping/ordering: real API sorts `todo, journal, asset, recipe` then lexically by
  name (`search.py:_sort_key`) — group order should match mockup's
  journal→assets→recipe ordering coincidentally, but a `todo` group hit (our fixture has one,
  from `todos.md`'s "token file" checkbox text) will appear FIRST and has no mockup
  counterpart — don't flag its presence as a bug, it's real content the mockup simply didn't
  anticipate.

### asset-image — `/assets/2026-07-14/architecture-sketch.png`

- Placeholder art (mockup's diagonal-hatch pattern icon box) vs a real rendered PNG
  (`gradient:'#1a1a2e-#0f3460'`, 1920×1080) — always exclude actual pixel content, compare only
  frame/border/shadow/lightbox chrome.
- File info panel: "Size" (812 KB mockup vs our fixture's real ~16.6KB), "Dimensions" (should
  actually match: 1920×1080), "Linked from" (no backing feature — exclude row entirely).

### asset-text — `/assets/2026-07-14/deploy-notes.txt`

- Shiki token colors vs mockup's `.tok-*` spans — same palette-only comparison rule as journal.
- Toolbar text: real is `{watch_path} · {lineCount} lines · {lang}` — `lineCount` will read 15
  (14 mockup content lines + 1 modeline line we added for language detection), never the
  mockup's fictional "52 lines"; `lang` should resolve to `bash` via the modeline. Ignore the
  literal numbers, compare only that the toolbar band exists/is styled consistently.
- "Copy file" labeled button (mockup) vs unlabeled floating "Copy" button (real,
  `copyButtons.ts`) — different affordance entirely, ignore button *label* text.
- File info card "Size" (1.4 KB mockup vs our real ~250 bytes) and "Linked from" (no backing
  feature) — exclude both.
- The extra modeline line itself (`# vim: ft=bash.`) will be visible as line 1 of the rendered
  file — the mockup has no such line. Expected, not a bug.

### asset-drawio — `/assets/2026-07-14/request-flow.drawio`

- Canvas: mockup shows a static dashed-grid placeholder + caption; real viewer (if/when wired)
  would render actual mxgraph content from our minimal 4-node/4-edge XML — exclude canvas
  content, compare only canvas frame/toolbar chrome (Open in draw.io / Download buttons).
- File info: "Size"/"Shapes" counts are decorative-only in the mockup with no real equivalent
  surfaced by the API today — exclude.

### download — `/assets/2026-07-12/backup.tar.gz`

- File size ("4.1 MB" mockup vs our real 198-byte tarball) and "Modified" timestamp — exclude,
  compare only card layout/icon/button chrome.

### 404 — `/definitely-not-a-page`

- Mockup's specific copy ("journal/2026/07/2026-07-31.md doesn't exist...") is a canned example
  for a *guessed* journal-shaped URL; our probe URL (`/definitely-not-a-page`) doesn't parse as
  a journal date at all, so the real `NotFound.svelte` message text will differ ("no route in
  client's matcher table" case, not the "journal day not written yet" case per its own source
  comment). Exclude message *text*, compare only layout/glyph/button chrome.
- `ws-dot is-down` state shown in mockup — exclude per the global ws-dot rule; real connection
  state at shot time is whatever the live socket reports.

### recipe-audit (unpaired) — `/recipes/sourdough-starter`

- No mockup exists; audit only for token consistency against the journal doc-layout bands
  (headings, tables, list spacing, deco-rule, breadcrumbs). Not a pixel-diff pair — no
  band-by-band exclusion list applies, just a general consistency sweep.

### Global exclusions (apply to every page)

- `ws-dot` pulse/glow animation and its live/reconnecting/down *state* — harness disables all
  animation/transition CSS and connection state is nondeterministic at shot time regardless.
- Any file-size/mtime/line-count/word-count metadata text anywhere (dir listings, asset info
  panels, asset-text toolbar) — none of it is backed by matching real values in this fixture
  (deliberately not chased byte-for-byte), and several mockup pages invent metadata about files
  that don't exist in this form on disk.
- Real Shiki syntax-highlight token colors vs mockup's hand-authored `.tok-*` spans — compare
  palette intent (keyword/string/comment/constant hues), never exact token boundaries.
