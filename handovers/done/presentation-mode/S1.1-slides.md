# presentation-mode / S1.1 slides.ts

**Responsibility:** Pure, DOM-touching-but-side-effect-free helpers that turn a rendered
`.markdown-body` element into a list of per-h1 slide HTML strings, and that compute
slide-index stepping / arrow dimming for presentation-mode navigation.

**Boundary:** `server/frontend/src/lib/presentation/slides.ts` (new file, new dir) and
`server/frontend/src/lib/presentation/slides.spec.ts` (new). Nothing else touched.

## What downstream needs from me

- `splitSlides(root: Element): string[]`
  - Walks **direct children only** of `root` (`root.childNodes`, including text nodes).
  - Starts a new slide at each top-level `<h1>` element.
  - Each slide string is the concatenation of `outerHTML` for element children and
    `textContent` for text/comment children, in document order.
  - Preamble (content before the first h1): included as slide 0 **only if** it has
    non-whitespace content; a whitespace-only preamble (e.g. formatting newlines between
    block elements) produces no extra slide.
  - Zero `<h1>`s anywhere → the whole content comes back as a single slide (array length 1),
    as long as it's non-empty.
  - An `<h1>` nested inside another top-level element (blockquote, details, etc.) is
    **not** a boundary — it's baked into that top-level element's `outerHTML` and never
    inspected directly, so this falls out of the "direct children only" walk for free.
  - Gotcha: a slide that *does* start with an h1 is always pushed, even if you construct
    a degenerate case where that seems empty — the h1's own outerHTML is never empty, so
    this doesn't arise in practice.

- `step(index: number, count: number, dir: 1 | -1): number`
  - Returns `index + dir`, clamped to `[0, count - 1]`. No wraparound at either edge.

- `arrowOpacity(index: number, count: number, dir: 1 | -1): number`
  - `0.5` if `index + dir` is a valid slide index (i.e. a slide exists in that direction),
    else `0.1`. With `count === 1`, both directions are `0.1`.

All three are pure functions with no module-level state; nothing to initialize or tear down.
