# server-rewrite / T25.3 ‚Äî route views

## Responsibility

Wires the SPA's actual page content: every `src/routes/*.svelte` (except
`Search.svelte`, S25.4's) now renders a real `src/lib/components/*Page.svelte`
implementation instead of the T25.1 placeholder stub, fetching from `api.ts`
and reusing the S25.2 enhance/MarkdownView infrastructure. Also makes
`App.svelte`'s breadcrumb trail real, and vendors a self-hosted drawio viewer
for `kind === "drawio"` docs.

**Boundary touched:** `server/frontend/src/{routes,components,lib}` (routes/
rewritten, App.svelte's breadcrumb wiring, one small `app.css` addition for
`.markdown-body` typography ‚Äî see Deviations #1) + `server/frontend/public/vendor/drawio/`
(new). No backend, no docs.

## Page set (`src/lib/components/`)

- **`DirPage.svelte`** ‚Äî `/` and `/dir/*`. Journal-month directories (every
  non-dir entry is a `YYYY-MM-DD` day file) render week-banded rows
  (`weekBands.ts`, below) inside the existing `.week-band` zebra-striped
  divs; everything else is a plain `.row` list. Per-entry href is derived
  from `DirEntry.doc_type` (journal ‚Üí `/journal/:date` or `/todo` for
  `journal/todos.md`; asset ‚Üí `/assets/:date/:file` reconstructed from the
  fixed `assets/YYYY/MM/DD/name` relpath shape; recipe ‚Üí `/recipes/*`;
  directories always ‚Üí `/dir/{relpath}`).
- **`JournalPage.svelte`** ‚Äî `/journal/:date`. `.layout-with-rail`/`.rail`
  (already in app.css, T22 item 7 zero-drift sticky spec) with the TOC
  (`{@html doc.toc}` ‚Äî Python-Markdown's `TocExtension` output already
  carries a `class="toc"` div, so it slots straight into the existing
  `.toc`/`.toc a` rules) + prev/next `.day-nav` from `DocPayload.nav` +
  `MarkdownView`.
- **`TodoPage.svelte`** ‚Äî `/todo`. `/api/todo` calls the same
  `build_doc_payload` as any doc, so `toc` **is** populated (unlike the
  legacy Flask route) ‚Äî this gets the same TOC rail as JournalPage, just
  no day-nav. Plain `MarkdownView`, no structured Overdue/Today/Upcoming
  grouping (see Deviations #2).
- **`DocPage.svelte`** ‚Äî kind-dispatch for any other single file (recipe
  files, non-day assets). Fetches `getDoc`, shows "‚Üê Back to journal/‚Ä¶"
  from `journal_date` when present, then dispatches on `doc.kind`:
  - `"text"` ‚Üí **`TextFileView.svelte`** ‚Äî builds a synthetic
    `<pre><code class="language-X">` and calls the *same*
    `enhance/shiki.ts` `highlightBlock()` + `enhance/copyButtons.ts`
    `addCopyButton()` used by markdown code blocks. Files over 500KB
    (`new Blob([text]).size`) skip `highlightBlock()` but still get the
    plain escaped `<pre>` + copy button.
  - `"image"` ‚Üí **`ImageView.svelte`** ‚Äî inline `<img>`, dimensions from
    the `load` event's `naturalWidth`/`naturalHeight`, fullscreen via the
    shared `enhance/lightbox.ts` `openLightbox()` singleton (not the
    mockup's CSS-only lightbox ‚Äî the Svelte one already exists).
  - `"drawio"` ‚Üí **`DrawioView.svelte`** ‚Äî see vendor section below.
  - `"binary"` ‚Üí **`DownloadCard.svelte`** ‚Äî path + direct
    `rawUrl(watch_path, {download:true})` link, per `mockups/download.html`.
  - `"markdown"` ‚Üí plain `MarkdownView`.
- **`EmptyState.svelte`** ‚Äî generic 404/error card (`mockups/404.html`
  markup, all-existing `.empty-state`/`.empty-glyph` classes). Used by
  every page's not-found branch and the SPA catch-all `NotFound.svelte`.

## Pure logic ‚Äî `src/lib/weekBands.ts` (+ `weekBands.spec.ts`, 15 tests)

`isJournalDayName`, `isoWeekNumber` (standard nearest-Thursday ISO-8601
algorithm), `bandByWeek`. Fixtures cross-checked against Python's
`date.isocalendar()` for year-boundary cases (2018-12-31‚Üí2019-W01,
2020-12-31‚Üí2020-W53, 2016-01-04‚Üí2016-W01, 2017-01-01‚Üí2016-W52) plus July
2026. Keys bands by bare week number (no ISO-year) ‚Äî safe because callers
only ever pass one directory listing's entries (one calendar month), where a
Dec/Jan wraparound always produces distinct week numbers.

## Breadcrumbs ‚Äî `src/lib/breadcrumbs.svelte.ts`

`breadcrumbs` (a `$state` store `App.svelte` reads), `withCurrent(base, name,
target)` for doc/journal pages (their `breadcrumbs` payload excludes the
current file per backend `include_cur_dir=False`; dir payloads already
include themselves), and `fallbackCrumbs(path)` for pre-fetch/404 states
(one crumb per path segment, all targeting `/dir/...` ‚Äî safe because
`Breadcrumbs.svelte` never renders the *last* crumb as a link, only as
plain text, so its `target` is cosmetic).

## Recipes route disambiguation

`/recipes/*`'s `rest` param doesn't distinguish a directory from a file, but
this resolved cleanly from the backend contract rather than needing a
heuristic: `make_breadcrumbs` always targets `/dir/...` for directories
(confirmed in `content.py`), and `server.old/app.py`'s legacy
`/recipes/<path>` route only ever rendered a content file. So `/recipes/*`
is *always* a file dispatch (`DocPage`) ‚Äî the only special case is the bare
`/recipes` root (empty `rest`), which has no file to show and falls back to
`DirPage(path="recipes")`. No dot-in-last-segment heuristic needed. A
manually-typed URL for an actual recipe *subdirectory* (e.g. `/recipes/bread`
typed by hand rather than reached via a generated `/dir/recipes/bread` link)
will 404 through `DocPage` ‚Äî accepted, since the app itself never generates
such a link.

## Vendor pin ‚Äî `public/vendor/drawio/`

- **`viewer-static.min.js`** (3,969,199 bytes, sha256
  `7f736ca32f5601fa9ae16b09965c1e6e89bff2381b0ba4ea79510ddf9f8d73c8`) from
  `jgraph/drawio` tag **`v30.3.11`**, fetched from the source tree
  (`src/main/webapp/js/viewer-static.min.js` at that tag) rather than the
  release asset ‚Äî the GitHub release only ships a 52MB `draw.war`, no
  standalone JS file; same commit/build either way.
- Full rationale, usage contract, and the residual `PROXY_URL`/etc.
  external-URL risk note live in `public/vendor/drawio/README.md` ‚Äî read
  that before touching `DrawioView.svelte`.
- `DrawioView.svelte` lazy-injects it as a classic `<script>` tag (singleton,
  `<script module>` top-level promise) from
  `${import.meta.env.BASE_URL}vendor/drawio/viewer-static.min.js`, renders
  `<div class="mxgraph" data-mxgraph='{"xml":...,"lightbox":false}'>`, and
  calls `window.GraphViewer.processElements()` itself after every `doc`
  change (the script only auto-runs once per page load, not per SPA
  navigation). `.gitattributes` already marks
  `server/frontend/public/vendor/**` generated/`-diff` (from T25.1).

## Deviations from the brief / mockups, assessed

1. **Added `.markdown-body` typography to `app.css`.** T25.2's handover
   explicitly flagged this as S25.3's job ("Page-level styling of
   `.markdown-body`‚Ä¶ is S25.3's job in app.css ‚Äî out of this unit's
   boundary"); ported verbatim from `mockups/journal.html`'s
   `.journal-body` rules, renamed to match the class `MarkdownView` actually
   emits. Small, required, not a "tidy-up".
2. **TodoPage has no Overdue/Today/Upcoming grouping or due-date badges**
   (`mockups/todo.html`). `/api/todo` returns a plain rendered
   `journal/todos.md` ‚Äî no structured task metadata, no due dates. Plain
   `MarkdownView` (with TOC, since one is genuinely populated) is what the
   real payload supports.
3. **DirPage shows only entries that exist on disk** ‚Äî no fabricated
   "no entry"/"future" placeholder day rows or word-count summaries
   (`mockups/dir-journal-month.html`'s demo content). The real
   `/api/dir` only returns `path.iterdir()` results.
4. **No file-size/dimensions/MIME info cards.** `DocPayload` doesn't carry
   them, and adding a client-side `HEAD` round-trip + `ETag`-parsing helper
   for a display-only nicety wasn't in the explicit task list ‚Äî skipped
   (ponytail: add a `FileInfoCard.svelte` + `api.ts` HEAD helper if this
   becomes an actual requirement). `ImageView` still shows dimensions
   (free ‚Äî from the `<img>` load event, no extra request).
5. **`DrawioView` drops the mockup's "Open in draw.io" button** ‚Äî that
   button would open `https://app.diagrams.net` with the diagram data,
   exactly the external-URL leak `lightbox:false` and self-hosting the
   viewer are meant to avoid. Only "Download .drawio" remains.
6. **No TOC scroll-spy** (highlighting the current section as you scroll,
   `.toc a.is-active` in app.css already has the style). Out of scope ‚Äî
   would need an `IntersectionObserver` pass; noted as a nice-to-have, not
   requested.
7. **No line-number gutter** in `TextFileView` (mockup's `.file-pre
   .file-line` hand-authored spans). Reusing `highlightBlock()` as designed
   (`codeToHtml` without a line-numbers transformer) was preferred over
   reimplementing highlighting or diverging from the shared enhance module;
   line numbers would need a Shiki transformer option not currently exposed
   by `shiki.ts`.
