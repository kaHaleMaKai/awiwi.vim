# server-rewrite / T25.2 — MarkdownView + enhance pipeline

## Responsibility

The client-side markdown enhancement layer for the SPA: a `MarkdownView`
component that injects server-rendered HTML via `{@html}` (no sanitization —
localhost-only, own notes) and an `enhance/` pipeline that runs post-render DOM
passes over it (Shiki highlighting, copy buttons, table export menus, checkbox
PATCH wiring, lazy mermaid, media rewrite + lightbox, redaction reveal). Plus
two pure helpers S25.3/S25.4 will also want: `lib/lang.ts` (Shiki language id
guessing, mirrors the server) and `lib/format.ts` (`beautifyDate`, pinned to the
Python server output).

**Boundary touched:** `server/frontend/src/lib/**` (enhance + helpers +
components) and `server/frontend/package.json` (two new deps). Also fixed one
scaffold bug in `src/lib/api.ts` (see Deviations). No routes/, no backend, no
docs.

## What S25.3/S25.4 consume

### `MarkdownView.svelte` (`src/lib/components/`)

The one component route views render for any `kind === "markdown"` doc.

```svelte
<script>
  import MarkdownView from "../lib/components/MarkdownView.svelte";
</script>
<MarkdownView
  html={doc.html}
  watchPath={doc.watch_path}
  onCheckboxStale={() => refetch()}
  onCheckboxSuccess={(mtimeNs) => { /* update tracked mtime */ }}
/>
```

Props:
- `html: string` (required) — `DocPayload.html`.
- `watchPath: string` (required) — `DocPayload.watch_path`; used for checkbox
  PATCH addressing and relative-image resolution.
- `onCheckboxStale?: () => void` — fired on a checkbox 409; the page should
  refetch the doc (the checkbox is auto-reverted first).
- `onCheckboxSuccess?: (mtimeNs: number) => void` — fired with the new
  `mtime_ns` after a successful toggle (feed your WS `mtime_ns` dedupe so the
  live-sync echo doesn't re-render).

Behaviour: re-runs `enhance()` whenever `html` or `watchPath` changes; follows
`theme.current` for the theme-dependent passes (mermaid) without re-highlighting
code. Renders a single `<div class="markdown-body">`. Page-level styling of
`.markdown-body` (heading margins, list indent, the `.journal-body` look from
the mockup) is S25.3's job in app.css — out of this unit's boundary.

### `enhance()` API (`src/lib/enhance/index.ts`)

```ts
enhance(container: HTMLElement, opts: EnhanceOptions): EnhanceHandle

interface EnhanceOptions {
  watchPath: string;
  theme: "dark" | "light";
  onCheckboxStale?: () => void;
  onCheckboxSuccess?: (mtimeNs: number) => void;
}
interface EnhanceHandle {
  setTheme(theme): void;   // re-runs mermaid; shiki switches via CSS, no re-highlight
  destroy(): void;         // detach listeners + unmount injected components
}
```

You normally don't call this directly — `MarkdownView` owns it. It's exported in
case a non-markdown view ever needs a subset. Idempotent-per-container: always
`destroy()` before dropping/re-rendering the container (MarkdownView does this
via its `$effect` cleanup).

### Module list (`src/lib/enhance/`)

- `index.ts` — orchestrator (`enhance`/`EnhanceHandle`, above).
- `shiki.ts` — `highlightBlock(pre)`. Lazy singleton highlighter
  (`import("shiki")` on first block), `github-light`/`github-dark` dual theme
  via `defaultColor:false`, switched by a one-time injected `[data-theme]` CSS
  rule (so a theme flip needs no re-highlight). Reads `textContent`. Unknown
  language → falls back to `text` (never throws). Replaces the server's
  `<pre><code class="language-x">` with Shiki's `<pre class="shiki">`.
- `copyButtons.ts` — `addCopyButton(pre)`. Wraps the block in
  `.awiwi-code-block` and floats a `.copy-btn`; copies `pre.textContent` at
  click. Injects its own positioning CSS.
- `tableExport.ts` — **pure**, unit-tested. `tableToRows`, `toMarkdown`,
  `toCsv` (RFC-4180 quote-escaping), `toHtml`, `serializeTable(table, fmt)`.
- `tables.ts` — `enhanceTables(container)`. Inserts a `.table-tools` bar above
  every `<table>` and mounts a `CopyMenu` into it (via Svelte `mount()`);
  cleanup `unmount()`s them.
- `CopyMenu.svelte` (`src/lib/components/`) — the table export dropdown
  (Markdown/CSV/HTML). Pick → serialize + clipboard + close + "Copied ✓" flash;
  click-outside closes without copying (per T22 feedback item 3).
- `checkbox.ts` — `wireCheckboxes(container, {watchPath, onStale, onSuccess})`.
  Reads `data-line-nr`/`data-hash` off `input.awiwi-checkbox`, PATCHes the new
  state; done-styling (`.awiwi-checkbox-done` strikethrough) follows the current
  state (T22 item 6); on 409 reverts + calls `onStale`; any error reverts.
- `mermaid.ts` — `renderMermaid(container, theme)`. Lazy `import("mermaid")` on
  first block; stashes source in `data-mermaid-src` before first render so a
  theme flip re-runs from pristine source; theme-aware (`dark`/`default`).
- `media.ts` — `resolveMediaSrc(src, watchPath)` (**pure**, tested) +
  `rewriteMedia(container, watchPath)`. Rewrites relative `<img src>` to
  `/api/raw/...` (mirrors the old `resolve_image_link`: `assets/…` is
  home-relative, else resolved against the doc dir; absolute/scheme/root URLs
  untouched) and wires click → lightbox.
- `lightbox.ts` — `openLightbox(src, alt)`. Lazily mounts one shared
  `Lightbox.svelte` on `<body>` (survives re-renders; only one ever exists).
- `Lightbox.svelte` (`src/lib/components/`) — fullscreen overlay (`.lightbox-*`
  markup from app.css). Closes on overlay click / close button / Escape.
- `redaction.ts` — `wireRedaction(container)`. Click/Enter/Space toggles
  `.is-revealed` + `aria-pressed` + title on `.redacted` spans/divs (T22 item 4).
- `clipboard.ts` — shared `writeClipboard` (best-effort) + `flashCopied`.

### `lib/lang.ts`

```ts
guessLanguage(path: string, text?: string | null): string | null
modelineLanguage(text: string): string | null
EXT_LANG_MAP: Record<string, string>
```

Hand-mirrors `mdrender.guess_language` (T23.3): vim `ft=` modeline (with the
`pgsql → sql` alias) wins over filename; `Dockerfile*` by prefix; else the
lowercase extension via `EXT_LANG_MAP`; `null` otherwise. **Keep in lockstep
with `server/src/awiwi/mdrender.py`'s `_EXT_LANG_MAP`/`LEXER_MAP` if either
grows.** S25.3's `TextFileView` will want this to pick the Shiki lang for
whole-file highlighting of `kind === "text"` docs (though `DocPayload.language`
is already server-computed — use that when present; `guessLanguage` is the
client-side fallback / cross-check).

### `lib/format.ts`

```ts
beautifyDate(value: string | Date, format?: string | null): string
```

Pinned byte-for-byte to the Python server's `templating.beautify_if_date`
(C-locale weekday/month names), **including the `<sup>` ordinal markup and the
historical 11st/13rd quirk** (suffix keys off the last digit only). Non-dates
pass through unchanged. Journal titles use `"%B"` / `"%B %Y"`; dir entries pass
no format. If you need the plain-text form (as the legacy `journal.html.j2` did
with `| striptags`), strip `<sup>…</sup>` yourself — `beautifyDate` deliberately
keeps it.

## Deps added (`package.json` → `dependencies`)

- `shiki@^3` (`^3.23.0`) — dynamic-imported.
- `mermaid@^11` (`^11.16.0`) — dynamic-imported.

Both are `import()`-only (plus a type-only `import type` in shiki.ts), so
they're code-split into lazy chunks. **They don't appear as separate chunks in
`npm run build` yet** because nothing imports `MarkdownView` in the current
graph — it's tree-shaken out. Once S25.3 wires `MarkdownView` into a route view,
the shiki/mermaid chunks materialize automatically. This is expected, not a bug.

## Gotchas / notes

- **No sanitization** on `{@html}` — deliberate (localhost, own notes). Record
  the ADR in S27.2 (plan lists it).
- **Shiki dual-theme CSS** is injected by `shiki.ts` at runtime (into
  `<head>`), not in app.css — app.css was out of this unit's boundary. If you'd
  rather own it in app.css later, the injected rule is small (see
  `injectThemeSwitchCss`). Same for the code-block/table-tools/checkbox-done
  micro-styles (each injected by its module). If app.css grows canonical
  versions, delete the injections.
- **CopyMenu/Lightbox are mounted imperatively** (`mount()`/`unmount()` from
  svelte) because they live inside `{@html}` content that Svelte doesn't manage.
  `enhance().destroy()` unmounts the CopyMenus; the Lightbox is an app-lifetime
  singleton (never unmounted — fine).
- **Checkbox `onSuccess(mtime_ns)`**: wire this into S25.4's WS `mtime_ns`
  dedupe, or a user's own toggle will look like a remote change and trigger a
  needless re-render (losing the enhanced DOM/scroll).
- **`renderMermaid` mutates the DOM** (replaces div content with `<svg>`); the
  `data-mermaid-src` stash is what makes re-theming possible. Don't strip that
  attribute.
- Shiki/mermaid/checkbox/media/lightbox aren't unit-tested (they need a real
  browser + WASM / network). The **pure** logic is: `tableExport`, `lang`,
  `format`, `resolveMediaSrc`, and `redaction` (DOM-level, happy-dom). 53 specs
  total.

## Deviations from the brief / scaffold, assessed

1. **Fixed `api.ts` `CheckboxPatchRequest.line_nr` → `line_no`.** The scaffold
   (T25.1) named this field `line_nr`, but the frozen contract
   (T23.2-api-routes.md) and the backend `CheckboxPatchBody` both require
   `line_no` — a request with `line_nr` would 422. `api.ts` is in this unit's
   boundary and the checkbox wiring depends on it, so it was corrected (comment
   left in place). S25.4, if it also PATCHes checkboxes, uses `line_no`.
2. **CSS injected from enhance modules** rather than added to app.css (out of
   boundary). See gotchas.
3. **`MarkdownView`/`CopyMenu`/`Lightbox` live in `src/lib/components/`**, per
   the T25.1 scaffold convention ("reusable components in `src/lib/components/`"),
   which the brief's `{lib,components}` boundary phrasing allows. No files under a
   top-level `src/components/`.
4. **`beautifyDate` accepts only strict `YYYY-MM-DD`** ISO strings (plus `Date`
   objects); exotic `date.fromisoformat` forms (week dates, basic format) aren't
   parsed and pass through unchanged. Our inputs are journal dates and dir names,
   so this matches server output for every real case.
