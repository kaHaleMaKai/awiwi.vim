# T23.1 S23.1 — schemas + payload builders + httputil (JSON API foundation)

## Responsibility

Additive foundation for a future Svelte SPA JSON API: Pydantic response
models (`schemas.py`), payload builders that construct them from the
existing pure leaf modules (`docs.py`), and a small httputil relocation
(`httputil.py`) so `docs.py` doesn't need to import Jinja machinery to get
`is_localhost`. **Nothing here is wired into an HTTP route yet** — no
`/api/*` endpoints exist. That's S23.2's job (the router unit this handover
is written for).

## Boundary

Touched only:

- `server/src/awiwi/schemas.py` (new)
- `server/src/awiwi/docs.py` (new)
- `server/src/awiwi/httputil.py` (new)
- `server/src/awiwi/templating.py` (relocated `is_localhost`/`get_home` out,
  re-exported for backward compat — see below)
- `server/tests/test_api.py` (new)

**Not touched:** `routers/pages.py`, `routers/assets.py`, `routers/actions.py`,
`content.py`, `mdrender.py`, `checkbox.py`, `search.py`, `conftest.py`, or any
pre-existing test file. Verified via `git status`/`git diff` — zero drift.
The old template routes and their tests are byte-for-byte untouched.

## Important: the "moved, not rewritten" boundary conflict

The original design contract for this subtask said `render_content_file`'s
and `_build_dir_index`'s (both in `routers/pages.py`) logic should be
*extracted* so the old template route and the new JSON builders share one
implementation. The concrete task boundary I was given only permits touching
`routers/pages.py` to import relocated helpers (not to restructure its
private functions), and requires the old routes to stay behaviorally
untouched. These two instructions conflict, and the hard boundary won:

**`docs.py` independently re-implements the same dispatch logic** (kind by
extension/mime, the secret-file regex gate, breadcrumb construction, nav
computation) rather than importing/sharing `pages.py`'s private
`render_content_file`/`_build_dir_index` functions. It's a faithful,
carefully cross-checked re-implementation — same regex (`_SECRET_RE`), same
call sequence into `content.py`/`mdrender.py`, same field derivations — not
an approximation. `test_api.py` pins several of these against the same
fixture data `test_acceptance.py` already exercises, so any future drift
between the two implementations would surface as a test failure in at least
one of the two suites.

**Whoever eventually does the real shared-helper extraction** (a cleanup
subtask, not blocking anything today) should: pull the kind-dispatch
if/elif chain and the secret-gate check out of both
`render_content_file` and `build_doc_payload` into a small shared function
in `content.py` (in-boundary for that future subtask, not this one), and
similarly for `_build_dir_index`'s per-entry doc-type/naming logic vs.
`build_dir_payload`'s. Until then, **if you change `render_content_file`'s
dispatch or secret-gate behavior, check `docs.py`'s copy for the same
change** (and vice versa) — they are two independent implementations of one
behavioral contract, not one shared implementation.

## What S23.2 (and beyond) needs

### `schemas.py` — Pydantic models

```python
from awiwi.schemas import (
    DocKind, DocType,               # Literal type aliases
    BreadcrumbPayload, NavPayload,  # small supporting models
    DocPayload, DirEntry, DirPayload, SearchHit,
)
```

- `DocKind = Literal["markdown", "text", "image", "drawio", "binary"]`
- `DocType = Literal["journal", "asset", "recipe", "other"]`
- `BreadcrumbPayload(name: str, target: str)` — JSON-serializable copy of
  `content.Breadcrumb` (which stays a plain dataclass; not itself in this
  subtask's boundary).
- `NavPayload(prev: str | None, next: str | None)` — mirrors
  `content.get_prev_and_next_journal`'s return shape.
- `DocPayload` — see field table below.
- `DirEntry(name: str, relpath: str, is_dir: bool, doc_type: DocType)`,
  `DirPayload(breadcrumbs: list[BreadcrumbPayload], entries: list[DirEntry])`.
- `SearchHit(target: str, name: str, line: int, col: int, type: str, text: str)`
  — JSON-serializable copy of `search.SearchHit` (the dataclass). **No
  builder produces this yet** — `docs.py` has no search-related function.
  S23.2's search router can construct these directly from
  `search.parse_search_output`/`sort_hits` results field-by-field (or
  `SearchHit(**dataclasses.asdict(hit))`).

### `DocPayload` field semantics

| field | type | meaning |
|---|---|---|
| `kind` | `DocKind` | content-shape dispatch result (see table below) |
| `doc_type` | `DocType` | top-level dir classification (`journal`/`asset`/`recipe`/`other`) |
| `html` | `str \| None` | rendered markdown body, `kind=="markdown"` only |
| `toc` | `str \| None` | `mdrender.RenderedDoc.toc` passthrough (HTML snippet, not a parsed tree — see divergence below), `None` if empty or non-markdown |
| `text` | `str \| None` | raw text (`kind=="text"`) or raw XML (`kind=="drawio"`); `None` otherwise |
| `language` | `str \| None` | best-effort Pygments lexer alias, `kind=="text"` only; `None` is a normal, expected outcome |
| `raw_url` | `str \| None` | `/api/raw/{watch_path}` for `kind in ("image","binary")`; **this route doesn't exist yet** — it's a URL string only, S23.2 (or later) must implement `GET /api/raw/{relpath:path}` to actually serve it, **with its own independent secret-gate check** (see Security below) |
| `watch_path` | `str` | home-relative **POSIX** relpath (`Path.as_posix()`), e.g. `"journal/2026/07/2026-07-01.md"`. This is the doc's stable identity: future WS subscription key AND future checkbox-PATCH `path` field. **Divergent from today's shipped `PATCH /checkbox` contract**, which takes a page *endpoint* string (e.g. `"/journal/2026-07-01"`, mapped back to a file via `actions.py:_file_for_endpoint`) — the new JSON API's checkbox endpoint (whenever built) should accept `watch_path` directly and skip that endpoint→file indirection entirely. |
| `breadcrumbs` | `list[BreadcrumbPayload]` | same trail `make_breadcrumbs` produces for the template routes |
| `journal_date` | `str \| None` | asset docs only: ISO date of the owning journal day (`"YYYY-MM-DD"`), enables "back to journal"; `None` for every other doc_type |
| `nav` | `NavPayload \| None` | journal-day pages only (doc_type `"journal"` whose filename stem parses as an ISO date); `None` otherwise, including for `journal/todos.md` |
| `mtime_ns` | `int` | `Path.stat().st_mtime_ns` at build time |
| `is_secret` | `bool` | filename stem matches `\b(secret\|credential)s?\b$` |

### Kind dispatch table (`build_doc_payload`)

Checked in this order (mirrors `render_content_file`'s dispatch, reordered
per the original design contract's literal wording — the two orders never
actually produce different results since the categories don't overlap in
practice):

1. `.md` extension → `"markdown"`
2. mime type (guessed from extension via `mimetypes.guess_type`) starts
   with `"image"` → `"image"`
3. `.drawio` extension → `"drawio"` (content = raw XML text)
4. UTF-8-decodable → `"text"` (content = raw text + best-effort language
   hint)
5. else → `"binary"` (no text/html, only `raw_url`)

Extension-less files fall into bucket 4 or 5 depending on decodability —
there's no special-case branch for them (unlike
`render_content_file`, which has an explicit "no extension → raw text"
early return; same *outcome*, different code path).

### Builder signatures

```python
from awiwi.docs import build_doc_payload, build_dir_payload, build_journal_payload

def build_doc_payload(path: Path, home: Path, *, is_localhost: bool) -> DocPayload: ...

def build_journal_payload(
    date_str: str, home: Path, *, is_localhost: bool, today: date | None = None
) -> DocPayload | None: ...

def build_dir_payload(dirs: str, home: Path) -> DirPayload: ...
```

- **`build_doc_payload`** — arbitrary file under `home`. Raises
  `FileNotFoundError` if `path` doesn't exist (propagated from `Path.read_*`
  calls) — a future router should map that to 404, same as the app's
  existing `FileNotFoundError` exception handler does for template routes.
  Passes `title=path.stem` to `render_markdown` for markdown files — this
  means a leading `# H1` line **stays in the rendered `html` body** (matches
  `render_content_file`'s behavior for recipes/assets/arbitrary `.md`
  files).
- **`build_journal_payload`** — the `/journal/{date_str}` JSON equivalent.
  Runs `date_str` through `content.parse_date` (so `today`/`yesterday`/
  `prev`/`previous` aliases resolve identically to the template route);
  `today` param exposed for deterministic tests. Returns `None` when the
  date string doesn't parse (including an exhausted `prev` alias) — map to
  404. Raises `FileNotFoundError` if the date is well-formed but no journal
  file exists for it (same as the `journal` route). **Does NOT pass a title
  override** to `render_markdown` — a leading `# YYYY-MM-DD` H1 is extracted
  as the title and excluded from `html` (matches the `journal` route
  exactly, and is a genuine, deliberate difference from
  `build_doc_payload`'s generic dispatch — not an inconsistency to "fix").
- **`build_dir_payload`** — `dirs` is the same `{dirs:path}` string shape
  the `/dir/{dirs:path}` route parses (empty string = home root). Never
  raises for a missing directory in the same way the old route would (both
  call `path.iterdir()` unguarded — a nonexistent `dirs` value raises
  `NotADirectoryError`/`FileNotFoundError` either way, unchanged).

## Security — secret-file gate

`_SECRET_RE = re.compile(r"\b(secret|credential)s?\b$")` applied to the
file's stem (verbatim copy of `routers/pages.py:_SECRET_RE` — duplicated,
not shared, per the boundary conflict above).

When `is_secret and not is_localhost`: **every content-bearing field is
blanked to `None`** — `html`, `toc`, `text`, `language`, **and `raw_url`**.
The design contract's literal wording only called out `html`/`text`, but I
extended the blanking to `raw_url` and `toc`/`language` too: an un-blanked
`raw_url` would hand the SPA a working link to the secret bytes despite
`is_secret=True` on the same payload, which defeats the stated goal ("the
SPA is not a trust boundary"). `kind`, `doc_type`, `watch_path`,
`breadcrumbs`, `journal_date`, `nav`, `mtime_ns`, `is_secret` are **not**
blanked — none of that is sensitive content, and dir listings already show
secret filenames unfiltered (matches existing, unchanged legacy/T16
behavior — see T16 handover divergence #6).

**S23.2 must independently re-check secrecy** in whatever eventually serves
`GET /api/raw/{relpath}` — a `DocPayload` with `raw_url` populated is not
itself a security control over that future endpoint; the endpoint must
re-derive `is_secret` (or receive it) and refuse to serve secret bytes to
non-localhost callers on its own, the same way `render_content_file` gates
content inline today.

On localhost, secret files render normally (`is_secret=True` but content
populated) — matches legacy/current behavior exactly (secret gate is a
network-trust gate, not an always-on redaction).

## `toc` divergence flagged

The design contract's field description says `toc` is "structure from the
markdown TOC extension." What's actually available from `mdrender` is
`RenderedDoc.toc`, an HTML snippet string (`<div class="toc">...</div>`),
**not** a parsed tree structure — that's all `mdrender.py` (T15, not in this
subtask's boundary) produces. `DocPayload.toc` is a straight passthrough of
that string (or `None` when empty/non-markdown). If the SPA needs a real
parsed heading tree (`[{level, id, text}, ...]`) rather than an HTML
fragment to render its own TOC widget, that requires either a change to
`mdrender.py` (out of boundary here) or client-side parsing of the HTML
snippet. Flagging for whoever picks up the SPA's TOC UI.

## What S23.2 needs to know (summary)

1. Import `DocPayload`/`DirPayload`/`SearchHit`/etc. from `awiwi.schemas`;
   call `build_doc_payload`/`build_journal_payload`/`build_dir_payload` from
   `awiwi.docs` to construct them.
2. Map `None` returns from `build_journal_payload` and `FileNotFoundError`
   from any builder to 404, same as the existing app-wide handler.
3. `raw_url`/watch_path-based checkbox PATCH/websocket-subscription
   endpoints don't exist yet — only the URL *shape* (`/api/raw/{relpath}`)
   and the *identity* (`watch_path`) are defined here. Building
   `GET /api/raw/{relpath:path}` (with its own secret-gate re-check) and a
   `watch_path`-keyed checkbox PATCH are S23.2's job.
4. `SearchHit` schema exists but no builder does — construct directly from
   `search.py`'s existing dataclass in the search router.
5. The `docs.py`/`routers/pages.py` duplication (see boundary-conflict
   section above) is intentional and load-bearing for now — don't "clean it
   up" by importing one from the other without re-reading that section.
